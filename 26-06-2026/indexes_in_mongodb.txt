}).sort({ joiningDate: -1 })


Important rule:

Left-prefix rule: this index helps department, department+status, department+status+joiningDate
It does not efficiently help status alone


3. Advanced Indexes

Multikey index (arrays):

db.employees.createIndex({ "projects.name": 1 })
db.employees.find({ "projects.name": "CRM Portal" })


Text index (search):

db.performance_reviews.createIndex({ comments: "text" })
db.performance_reviews.find({ $text: { $search: "excellent performance" } })

TTL index (auto-delete old docs):

db.sessions.createIndex({ expiresAt: 1 }, { expireAfterSeconds: 0 })


db.employee_documents.createIndex({ docNumber: 1 }, { unique: true })


Partial index (index subset only):

db.employees.createIndex(
  { department: 1, joiningDate: -1 },
  { partialFilterExpression: { status: "Active" } }
)

Sparse index (only docs containing field):

db.employees.createIndex({ tempAddress: 1 }, { sparse: true })


Wildcard index (dynamic fields):

db.employees.createIndex({ "meta.$**": 1 })

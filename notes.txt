Event Loop 

Event Loop is a contineous Operation, which checks for Async functionality , if Event Loop will get any Async functions it will attach a tread from thead Pool and then , CPU wil take care of the Task .


What is Nodejs ?

Nodejs is a Open Source(any one can use), Cross Platform (Any OS can use like windows, Linux, Mac etc.) Javascript Runtime Environment used for Building Server side Web applications(APIs, servers). 

Nodejs is Single threaded , Event Driven, Non Blocking I/O (Input output) architecture 

Nodejs is not a Programming Language, or not a framework, not a library .


What is Event Loop in Nodejs ?


Event Loop is the core functionality of the nodejs which executes the async functionality .if main thread encounters some async functionality it will offload to the Event loop , and event loop will handle and give appropriate Response.



Architecture of Event Loop .

1. MicroTask Queue ->  
    in micro task queue, two queues are there 
    1. Process.nexttick queue
    2. Promises Queue

2. Timer Queue 
    In timer queue two functions are there
    1. Settimeout 
    2. Setinterval

3. I/O Queue
    1. file Operation
    2. Network request

4. Setimmidiate Queue or Check queue

5. Closed Callback Queue
    socket.on('close')



1. what is the differnce between Process.nexttick and Setimmidiate ?
Process.nexttick will execute after this currenct executed program but Setimmidiate will execute after the next iteration of the event loop
